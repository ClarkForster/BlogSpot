# 从零搭一个长期可维护的技术博客

搭博客这件事，看起来像一个设计问题，但真正决定它能否长期存在的，往往是维护成本。

如果每发一篇文章都要碰数据库、后台管理、复杂部署流程，那么写作者很快就会把注意力从内容转移到系统本身。对个人博客来说，这通常不是好事。

## 我优先考虑什么

我给自己列了三个约束：

- 内容应该只需要 Markdown。
- 部署应该尽量自动。
- 风格应该服务阅读，而不是抢走文字的重心。

这也是为什么我最后选择了 VitePress。它不是功能最多的博客框架，但它足够简单，而且简单本身就是一种长期优势。

## 页面为什么要克制

我很喜欢两类网站：一种像技术文档，信息密度高；另一种像书页，留白充足，读起来不累。

这个博客尝试把二者折中：

- 首页做轻设计，只负责建立第一印象；
- 文章页则尽量退到后面，让正文成为主角；
- 颜色控制在淡蓝范围内，用来组织视觉层次，而不是制造噪音。

## 结语

如果一个系统想要被长期使用，它最好先学会不打扰使用者。博客也一样。
