# 博客完善清单

这份清单只记录**当前博客仍然值得完善的地方**。

在整理前，已经先检查了现有主题组件、样式、配置和内容结构，因此**不会把已经存在的功能重复写进来**。

## 已存在，不纳入本次清单

以下能力已经具备，本清单不再重复列为待办：

- 首页 Hero、精选文章、主题分组、最近更新时间线
- 文章列表页与 tag 筛选
- 独立页面布局与简历时间线
- 深色模式、响应式布局、动画过渡、reduced motion 兼容
- 标题优先的本地搜索
- 文章注释系统（侧栏 / 文内双模式）
- 页脚每日一言、复制反馈、路由切换刷新
- 代码块基础样式、图片居中与说明文字样式
- 基础 OG / favicon / sitemap / nav / outline / lastUpdated 配置

---

## P0：生产就绪与基础修正

### 1. 替换站点公开配置中的占位值
- [x] 将 `sitemap.hostname` 改为真实域名
- [x] 将 `socialLinks.github` 改为真实 GitHub 地址
- [ ] 复核全站 title / description / og 基础配置是否与博客定位一致

**为什么在清单里：** 这部分占位值已经完成替换，但仍需要继续复核全站对外展示文案是否足够贴合博客定位。

### 2. 检查站点对外展示信息的一致性
- [ ] 统一站点标题、描述、社交展示文案
- [ ] 确保首页、文章页、关于页在分享时不会出现泛化描述

**为什么在清单里：** 当前已经有基础 OG 配置，但仍偏全站级，表达粒度不够细。

---

## P1：阅读体验增强

### 3. 给代码块增加复制按钮
- [ ] 为文章代码块添加一键复制能力
- [ ] 补充复制成功提示，保持与页脚语录交互风格一致

**为什么在清单里：** 当前“点击复制”能力仅用于页脚每日一言，文章代码块仍未提供复制操作入口。

### 4. 给文章图片增加 lightbox / zoom
- [ ] 点击图片可放大查看
- [ ] 适配技术图表、截图与长图阅读

**为什么在清单里：** 当前图片已经有居中、圆角和阴影样式，但仍是静态展示。

### 5. 增加阅读进度条
- [ ] 在长文章页面显示阅读进度
- [ ] 控制样式尽量轻量，不抢正文注意力

**为什么在清单里：** 当前未见文章级滚动进度提示，长文阅读反馈还不够强。

### 6. 增加返回顶部按钮
- [ ] 页面滚动一定距离后显示
- [ ] 移动端也保持易点按

**为什么在清单里：** 当前未见独立 back-to-top 交互，对长文章回跳不够方便。

### 7. 统一文章详情页的 meta 信息展示
- [ ] 在文章页统一展示日期、分类、标签等元信息
- [ ] 与首页卡片、文章列表页保持一致的表达风格

**为什么在清单里：** 首页和列表页已经展示部分 meta，但文章详情页的统一 meta 条带还不明显。

---

## P2：内容发现与信息架构

### 8. 补齐 category 筛选闭环
- [ ] 让文章列表页真正支持 category 维度筛选
- [ ] 让首页已有的 category 入口和文章列表页联动起来

**为什么在清单里：** 首页已经有按 category 跳转的入口，但 `ArticleList.vue` 当前主要处理的是 tag 筛选。

### 9. 增加 archive / tag / category 独立索引页
- [ ] 提供文章归档页
- [ ] 提供标签聚合页
- [ ] 提供分类聚合页

**为什么在清单里：** 目前已有文章列表，但还缺少更完整的内容索引体系。

### 10. 提升搜索结果的信息密度
- [ ] 从 title-only 搜索逐步扩展到更丰富的结果信息
- [ ] 在结果中补充摘要、分类、日期等线索

**为什么在清单里：** 当前搜索已可用，但主要偏标题命中，正文发现能力较弱。

---

## P3：SEO 与内容分发增强

### 11. 从全站基础 SEO 升级到文章级 SEO
- [ ] 为文章补 canonical
- [ ] 补充更细粒度的 OG 信息
- [ ] 视情况增加结构化数据（如 Article schema）

**为什么在清单里：** 现在已有全站基础 metadata，但缺少文章级增强。

### 12. 规划默认分享图与文章分享图策略
- [ ] 准备默认分享图
- [ ] 明确文章页是否需要单独封面图 / OG 图

**为什么在清单里：** 当前有基础 `og:image`，但还未形成系统化分享图策略。

---

## P4：工程维护与长期演进

### 13. 拆分 `custom.css`
- [x] 将主题变量、首页、文章页、注释系统、简历页、footer 等样式拆分
- [x] 降低单文件维护成本

**为什么在清单里：** `custom.css` 已完成按职责拆分，但后续仍可继续围绕重复规则和覆盖关系做第二轮整理。

### 14. 梳理注释系统与布局常量的单一来源
- [ ] 减少 JS / CSS 中重复维护的布局常量
- [ ] 为后续注释系统继续增强打好基础

**为什么在清单里：** 文章注释系统已经很强，但结构上还有继续稳固的空间。

### 15. 为简历页补打印 / 导出支持
- [ ] 增加适合打印的样式
- [ ] 评估导出 PDF 或下载版简历的方式

**为什么在清单里：** 当前简历页浏览体验已经不错，但对投递/导出的支持还不明显。

---

## 建议执行顺序

如果要从最值得先做的几项开始，我建议顺序是：

1. 补齐 category 筛选闭环
2. 给代码块增加复制按钮
3. 给图片增加 lightbox / zoom
4. 继续增强文章级 SEO
5. 增加阅读进度条
6. 为简历页补打印 / 导出支持

---

## 使用说明

后续如果继续扩展这份清单，建议遵守两条规则：

1. **先确认功能是否已经存在，再决定是否加入清单**
2. **如果某项已有基础实现，就写成“增强/深化”，不要误写成“从零增加”**
